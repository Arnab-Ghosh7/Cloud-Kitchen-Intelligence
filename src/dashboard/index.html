Relational Database DDL (3NF)

PostgreSQL / SQLite Database Tables 
-- 1. Create Restaurants Table
CREATE TABLE restaurants (
    restaurant_id INTEGER PRIMARY KEY AUTOINCREMENT,
    restaurant_name VARCHAR(150) NOT NULL,
    locality VARCHAR(100) DEFAULT 'College Road',
    rating DECIMAL(2,1),
    number_of_reviews INTEGER,
    cost_for_two INTEGER,
    delivery_time_mins INTEGER,
    restaurant_type VARCHAR(50)
);

-- 2. Create Normalized Cuisines Table (1NF Compliance)
CREATE TABLE restaurant_cuisines (
    restaurant_id INTEGER,
    cuisine_tag VARCHAR(50),
    PRIMARY KEY (restaurant_id, cuisine_tag),
    FOREIGN KEY (restaurant_id) REFERENCES restaurants(restaurant_id)
);

-- 3. Create Menu Items Table
CREATE TABLE menu_items (
    item_id INTEGER PRIMARY KEY AUTOINCREMENT,
    restaurant_id INTEGER NOT NULL,
    item_name VARCHAR(200) NOT NULL,
    category VARCHAR(100),
    price DECIMAL(10,2),
    is_bestseller BOOLEAN,
    FOREIGN KEY (restaurant_id) REFERENCES restaurants(restaurant_id)
);

Analytical Queries Solutions

Assessment Solutions (Syntactically Correct) 
-- 1. Top 5 Highest Rated Restaurants
SELECT restaurant_name, rating, cost_for_two
FROM restaurants
ORDER BY rating DESC, number_of_reviews DESC
LIMIT 5;

-- 2. Average Cost-for-Two by Cuisine
SELECT c.cuisine_tag, ROUND(AVG(r.cost_for_two), 2)
FROM restaurants r
JOIN restaurant_cuisines c ON r.restaurant_id = c.restaurant_id
GROUP BY c.cuisine_tag
ORDER BY AVG(r.cost_for_two) DESC;

-- 3. Restaurants with More than 1 Cuisine Tag
SELECT r.restaurant_name, COUNT(c.cuisine_tag)
FROM restaurants r
JOIN restaurant_cuisines c ON r.restaurant_id = c.restaurant_id
GROUP BY r.restaurant_id, r.restaurant_name
HAVING COUNT(c.cuisine_tag) > 1;

-- 4. Highest Priced Menu Item
SELECT r.restaurant_name, m.item_name, m.price
FROM menu_items m
JOIN restaurants r ON m.restaurant_id = r.restaurant_id
WHERE m.price = (SELECT MAX(price) FROM menu_items);

Cloud Kitchen Feasibility & Market Study

Methodology, Assumptions, and Future Improvements

1. Data Collection & Simulation

The collection process gathered active listings in College Road, Nashik, utilizing coordinates lat: 20.0076, lng: 73.7634. We compiled a raw database of 37 listings, retaining duplicate records and null fields to serve as verified entries for data validation. Cleaned records reached a count of 35 unique, normalized restaurants.

2. Cleaning Strategies

Duplicate Detection: Executed an automated pipeline comparing normalized lowercase names within matching localities, removing redundancy.
Missing Values: Imputed ratings with the local dataset median of 4.2. Missing budget metrics for meals for two were resolved using the median budget of ₹350. Non-reviewed entries were assigned a count of 0.
Cuisine Normalization: Stripped formatting issues, resolved casing mismatches, and re-joined arrays in clean Title Case patterns.

3. Strategic Conclusions

Cuisine Saturation: Fast Food is hyper-saturated representing 34.3% of the local listing counts. Desserts and Cafe lines occupy another 48.6%. Price compression and massive advertising expenses are highly visible in these segments.
Feasible Venture: 'NutriBowl' (Healthy protein grains, custom macro bowls) operates in an underrepresented niche (occupying only 8.6% share) and offers higher customer loyalty and stronger initial margin support.
Operational Efficiency Benchmarks: South Taste acts as the efficiency leader, achieving a perfect rating of 4.6, 1450 reviews, a fast average delivery timeline of 15 minutes, and a pocket-friendly cost-for-two of ₹250. Standardized menus, rapid assembly prep, and high inventory turn rates underpin this benchmark performance.
